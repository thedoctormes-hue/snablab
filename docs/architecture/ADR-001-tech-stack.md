# ADR-001: Выбор технологического стека

**Дата:** 2026-05-15
**Статус:** Принято

## Контекст

Необходимо выбрать стек для системы управления закупками лабораторных расходных материалов. Система должна обрабатывать PDF/DOCX, иметь веб-интерфейс и масштабироваться.

## Решение

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| Backend | Python + FastAPI | Уже есть код парсера, async, быстрая разработка |
| Frontend | React + TypeScript + Vibe | Дашборды, типобезопасность |
| Database | PostgreSQL | Надёжность, JSONB, полнотекстовый поиск |
| Cache/Queue | Redis | Кэш справочников, сессии, очереди задач |
| Storage | MinIO (S3) | Хранение PDF КП, DOCX СЗ |
| PDF Parsing | pdfplumber + Tesseract OCR | Таблицы из PDF, сканированные КП |
| DOCX Gen | python-docx | Шаблонизация документов |
| Auth | JWT + bcrypt | Простота, безопасность |
| Deploy | Docker Compose | Локальный деплой на сервере |

## Альтернативы

- **Django вместо FastAPI**: избыточно для API-first архитектуры
- **Vue вместо React**: нет критических преимуществ для дашборда
- **MongoDB вместо PostgreSQL**: реляционные данные лучше ложатся на SQL

## Последствия

- Полная совместимость с существующим кодом парсера
- Асинхронная обработка запросов
- Типобезопасность на всех уровнях
