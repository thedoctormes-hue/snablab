# ADR-004: Парсинг коммерческих предложений

**Дата:** 2026-05-15
**Статус:** Принято

## Контекст

Поставщики присылают КП в формате PDF/DOCX. Необходимо автоматически извлекать структурированные данные: поставщик, позиции, цены, сроки.

## Решение

**Regex-based парсинг** текста, извлечённого из PDF/DOCX.

### Pipeline
```
PDF → pdftotext -layout → plain text → regex parser → structured data
DOCX → python-docx → plain text → regex parser → structured data
```

### Извлекаемые поля
- Поставщик: название, ИНН, адрес
- КП: номер, дата, получатель
- Позиции: наименование, каталожный номер, производитель, количество, цена, сумма, НДС
- Итоги: общая сумма, НДС, срок поставки, срок действия цен

### Ограничения
- Работает для «стандартных» форматов КП (КардиоМед, ИМБИАН ЛАБ и подобные)
- Сканированные PDF не поддерживаются (нужен OCR)
- Layout-dependent: при изменении формата КП нужна адаптация regex

## Альтернативы

| Подход | Плюсы | Минусы |
|--------|-------|--------|
| **Regex (выбрано)** | Быстро, просто, работает офлайн | Fragile к изменениям формата |
| pdfplumber tables | Лучше для табличных PDF | Не работает для сканов, сложнее |
| LLM-based extraction | Универсально | Дорого, медленно, нужен API |
| Tesseract OCR | Сканированные PDF | Медленно, неточный для таблиц |

## Последствия

- ✅ Работает офлайн, без внешних API
- ✅ Быстрый парсинг (< 1с на документ)
- ⚠️ Требует адаптацию под новые форматы КП
- 📋 План: добавить pdfplumber tables как fallback, LLM для сложных случаев
